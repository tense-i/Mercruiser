## 1. 文档目的

本 PRD 用于定义 Mercruiser 下一阶段的核心业务模型、生产工作流和产品范围。

本次重构的核心目标不是简单优化页面，而是将产品从“阶段式工作台”升级为“系列治理 + 单集工业化生产线”的双层体系。

## 2. 产品定义

Mercruiser 是一个面向 AI 短剧 / 漫剧生产的 Studio。

其本质不是单次生成工具，而是一个持续生产系统，包含：

- 系列治理层
- 单集生产线层
- Agent 编排层
- 任务与系统治理层

## 3. 产品定位

### 3.1 系列原生
产品以“系列”为顶层对象，以“集数”为交付单元，以“章节 / 镜头”为生产颗粒度。

### 3.2 工业化生产线
单集内部不是一个大而全工作台，而是一条明确生产线：

1. 概览
2. 剧本
3. 主体
4. 分镜
5. 故事板
6. 成片

### 3.3 Agent 编排驱动
Agent 不只是聊天工具，而是生产协调器，负责：

- 当前状态识别
- 缺口诊断
- 下一步建议
- 任务编排
- 失败恢复建议
- 质量提示

### 3.4 资产沉淀与复用
每一集生产不是孤立行为，高价值资产、风格规则、提示词经验必须沉淀回系列层。

## 4. 核心业务结构

### 4.1 系列层
负责：

- 系列基础信息
- 世界观 / 风格 / 导演规则
- 系列共享主体资产
- 系列级模型 / Prompt / Agent 策略
- 集数列表与总进度
- 系列级风险管理

### 4.2 集数层
负责：

- 单集源内容
- 单集章节结构
- 单集镜头结构
- 单集故事板
- 单集成片
- 单集导出结果

### 4.3 章节层
章节是剧本工位的工作单位，负责：

- 承载文本内容
- 控制字数与节奏
- 作为后续主体提取、分镜拆解的输入源

### 4.4 镜头层
镜头是分镜工位的工作单位，负责：

- 场景
- Prompt (提示词)
- 景别与构图
- 运镜 (Camera Motion)
- 光影
- 对白
- 音效
- 时长 (支持音频前置驱动的自动计算)
- 分镜描述
- Take (候选条) 管理
- 连戏状态 (Continuity Status)

### 4.5 故事板层
故事板是镜头的多模态装配层，负责：

- 镜头图
- 参考主体
- 分镜素材
- 镜头视频候选
- 选定版本

### 4.6 Agent 编排层
横跨全局，负责：

- 阶段推进
- 前置条件校验
- 推荐动作
- 批量动作触发
- 失败恢复建议

### 4.7 全局资产库

全局资产库是跨系列的资产管理层，负责：

- 存储可跨系列复用的角色、场景、道具
- 资产分类与检索
- 从系列资产升级到全局
- 从全局资产导入到系列

#### 全局资产数据结构

```typescript
interface GlobalAsset {
  id: string;
  name: string;
  type: 'character' | 'scene' | 'prop';
  ownerId: string;                      // 创建者
  
  // 基础资料
  description: string;
  prompt: string;
  referenceImages: string[];            // 多版本参考图
  selectedImageId?: string;             // 主版本图片
  voiceId?: string;                     // 绑定语音 (角色)
  
  // 一致性配置
  faceLocked: boolean;                  // 面部锁定 (角色)
  consistencyConfig?: {
    referenceStrength: number;          // 参考强度 0-1
    styleKeywords: string[];            // 风格关键词
  };
  
  // 使用统计
  usedInSeries: {
    seriesId: string;
    seriesName: string;
    linkedAssetId: string;              // 系列内资产ID
  }[];
  
  // 元数据
  tags: string[];
  createdAt: number;
  updatedAt: number;
}
```

#### 全局资产与系列资产关系

```
                    ┌─────────────────┐
                    │   全局资产库     │
                    │  GlobalAssets   │
                    └────────┬────────┘
                             │
           ┌─────────────────┼─────────────────┐
           │ 导入            │ 导入            │ 升级
           ↓                 ↓                 ↑
    ┌──────────────┐  ┌──────────────┐  ┌──────────────┐
    │   系列A资产   │  │   系列B资产   │  │   系列C资产   │
```

- **linked 资产**: 从全局导入，关联 `globalAssetId`，可选择保持同步
- **local 资产**: 仅限系列内使用，可升级为全局资产

## 5. Agent 驱动的生产工作流 (核心)

本章节详细定义 Agent 在各工位的具体职责、触发条件、输入输出和前置依赖。

### 5.1 工作流总览

```
小说原文导入
    ↓
[阶段1: 剧本生成] → ScriptAgent (手动触发)
    ├─ 输入: 小说原文 + 系列设定
    ├─ 输出: 按集数拆分的剧本章节
    └─ 产物: episodes[].chapters[]
    ↓
[阶段2: 主体提取] → AssetAgent (手动触发)
    ├─ 输入: 本集剧本章节
    ├─ 输出: 角色/场景/道具资产列表
    ├─ 产物: assets[] (含 name, description, prompt, type)
    └─ 门禁: 剧本完成后才可触发
    ↓
[阶段3: 资产生成] → 用户触发 (单个/批量)
    ├─ 输入: asset.prompt + 模型配置
    ├─ 输出: 资产图片 (多版本)
    ├─ 产物: asset.images[] + selectedVersion
    └─ 门禁: 提取完成后才可生成
    ↓
[阶段4: 分镜生成] → ProductionAgent (手动触发)
    ├─ 输入: 本集剧本 + 已生成资产列表
    ├─ 输出: 分镜表 (Shot List)
    ├─ 产物: shots[] (完整12字段结构)
    └─ 门禁: 本集所有资产必须已生成图片
    ↓
[阶段5: 分镜图生成] → 用户触发 (单个/批量)
    ├─ 输入: shot.prompt + 关联资产参考图
    ├─ 输出: 分镜图片 (多版本)
    ├─ 产物: shot.images[] + selectedVersion
    └─ 门禁: 分镜表生成完成
    ↓
[后续阶段: 故事板 → 成片 → 导出]
```

### 5.2 阶段1: 剧本生成 (ScriptAgent)

#### 5.2.1 工位概述

剧本工位是单集生产的起点，负责将原始文本（小说、故事大纲等）转化为结构化的剧本章节。

**工位布局**:
- **左侧**: 章节列表面板（可折叠）
- **中部**: 章节内容编辑区（主工作区）
- **右侧**: 全局设定侧边栏（视频比例、创作模式、风格参考等）
- **顶部**: 工位操作栏（导入原文、生成剧本、导出等按钮）

---

#### 5.2.2 场景1: 原文导入

##### 前置条件
- 用户已进入某一集的剧本工位
- 该集剧本状态为 `idle` 或 `draft`

##### 正常流程

**步骤1: 打开导入对话框**
1. 用户点击顶部操作栏的「导入原文」按钮
2. 系统弹出导入对话框，包含:
   - 文本粘贴区域（textarea，支持拖拽粘贴）
   - 文件上传区域（支持 .txt, .docx, .md）
   - 字数统计显示
   - 「确认导入」和「取消」按钮

**步骤2: 输入内容**
- **方式A - 粘贴文本**:
  1. 用户在文本区粘贴原文内容
  2. 系统实时显示字数统计: `已输入 12,345 字`
  3. 若超过上限，显示警告: `⚠️ 已超出单次导入上限 (50,000字)，请分批导入`

- **方式B - 上传文件**:
  1. 用户点击上传区域或拖拽文件
  2. 系统显示上传进度条
  3. 解析完成后显示: `✓ 已解析: xxx.docx (8,234 字)`
  4. 文件内容自动填充到文本区

**步骤3: 确认导入**
1. 用户点击「确认导入」
2. 系统将原文保存到 `episode.sourceText` 字段
3. 对话框关闭
4. 中部工作区显示原文预览，并出现「从原文生成剧本」按钮
5. 剧本工位状态 → `draft`

##### 异常流程

| 异常场景 | 触发条件 | 系统行为 | 用户可选操作 |
|---------|---------|---------|-------------|
| 文件格式不支持 | 上传 .pdf/.png 等 | 显示 Toast: `不支持的文件格式，请上传 .txt, .docx 或 .md 文件` | 重新选择文件 |
| 文件解析失败 | .docx 损坏或加密 | 显示 Toast: `文件解析失败，请检查文件是否损坏或加密` | 尝试粘贴文本 |
| 文件过大 | 文件 > 10MB | 显示 Toast: `文件过大 (>10MB)，请压缩后重试或分批导入` | 分批导入 |
| 内容为空 | 文本区为空时点击确认 | 按钮置灰，Tooltip: `请先输入或上传原文内容` | 输入内容 |
| 覆盖确认 | 已有原文时重新导入 | 弹出确认框: `当前已有原文内容，重新导入将覆盖。是否继续？` | 确认覆盖 / 取消 |

##### 边界条件
| 限制项 | 最小值 | 最大值 | 说明 |
|-------|-------|-------|------|
| 单次导入字数 | 100 字 | 50,000 字 | 超过需分批 |
| 文件大小 | - | 10 MB | 单文件限制 |
| 支持格式 | - | .txt, .docx, .md | 不支持 PDF |

#### 5.2.3 场景2: 剧本生成

##### 前置条件
- 原文已导入 (`episode.sourceText` 非空)
- 系列设定已配置 (世界观、风格) - **可选但推荐**
- 剧本工位状态为 `draft`

##### 正常流程

**步骤1: 触发生成**
1. 用户点击「从原文生成剧本」按钮
2. 按钮变为禁用状态，显示 Loading 图标
3. 系统弹出生成配置对话框:
   - 目标章节数: 自动计算 (原文字数 / 75)，可手动调整
   - 单章节字数: 50-100 字（默认 75）
   - 语言风格: 保持原文风格 / 精简提炼 / 口语化
   - 对白提取模式: 保留原文 / 自动补充 / 简化

**步骤2: 确认配置并开始生成**
1. 用户点击「开始生成」
2. 对话框关闭
3. 顶部出现生成进度条: `正在生成剧本... 分析原文结构 (1/4)`
4. 进度阶段依次显示:
   - `分析原文结构 (1/4)`
   - `拆分叙事节点 (2/4)`
   - `生成章节内容 (3/4)`
   - `提取对白与场景 (4/4)`
5. 剧本工位状态 → `generating`

**步骤3: 生成完成**
1. 进度条显示 100%，然后消失
2. 显示 Toast: `✓ 剧本生成完成，共 24 个章节`
3. 左侧章节列表刷新，显示所有生成的章节
4. 中部自动选中第一个章节，显示章节内容
5. 右侧显示章节元数据（场景、对白列表）
6. 剧本工位状态 → `completed`
7. 顶部出现「进入主体工位」引导按钮

##### Agent 职责详解

ScriptAgent 在生成过程中执行以下任务:

| 步骤 | 职责 | 输入 | 输出 |
|-----|------|------|------|
| 1. 内容分析 | 解析原文结构、识别叙事线 | sourceText | 叙事结构图 |
| 2. 节奏分析 | 识别高潮点、转折点、节奏变化 | 叙事结构 | 节奏标记 |
| 3. 章节拆分 | 按节奏标记拆分为独立章节 | 叙事结构 + 节奏标记 | chapters[] |
| 4. 对白提取 | 识别引号内容、说话人归属 | 各章节文本 | dialogues[] |
| 5. 场景标注 | 为每章节识别主场景 | 各章节文本 | scene 字段 |

##### 输出数据结构

```typescript
interface GeneratedScript {
  episodeId: string;
  generatedAt: number;           // 生成时间戳
  sourceTextHash: string;        // 原文 hash，用于变更检测
  chapters: {
    id: string;                  // UUID
    index: number;               // 章节序号，从 1 开始
    content: string;             // 章节文本 (50-100字)
    scene: string;               // 场景名称
    sceneDescription?: string;   // 场景补充描述
    dialogues: {
      id: string;                // 对白 ID
      speaker: string;           // 说话人
      content: string;           // 对白内容
      emotion?: string;          // 情绪标注 (可选)
    }[];
    wordCount: number;           // 字数统计
    isEdited: boolean;           // 是否被手动编辑过
  }[];
  metadata: {
    totalChapters: number;
    totalWords: number;
    uniqueScenes: string[];      // 去重后的场景列表
    uniqueSpeakers: string[];    // 去重后的说话人列表
  };
}
```

##### 异常流程

| 异常场景 | 触发条件 | 系统行为 | 用户可选操作 |
|---------|---------|---------|-------------|
| 生成超时 | API 响应 > 120s | 进度条变红，显示: `生成超时，请重试` | 重试 / 取消 |
| API 调用失败 | 网络错误 / 500 | 显示 Toast: `生成失败: [错误信息]` | 重试 |
| Token 超限 | 原文过长导致截断 | 显示警告: `原文过长，已截取前 30,000 字处理。建议分集导入。` | 继续 / 重新导入 |
| 内容不适合 | 原文为纯对话/诗歌等 | 显示: `原文结构无法识别为叙事内容，请检查格式` | 重新导入 / 手动编写 |
| 生成中断 | 用户点击取消 / 刷新页面 | 已生成部分不保存，状态回退到 `draft` | 重新生成 |
| 配额不足 | AI 调用额度用尽 | 显示: `AI 调用额度不足，请联系管理员充值` | 等待 / 手动编写 |

##### 生成中断恢复机制

当生成过程中发生中断（网络断开、页面刷新等）:

1. **前端行为**: 
   - 页面重新加载后检测 `episode.generationTaskId`
   - 若存在未完成任务，顶部显示: `检测到未完成的生成任务，是否继续？`
   
2. **用户选择**:
   - 「继续」: 恢复进度条，轮询任务状态
   - 「放弃」: 清除任务 ID，允许重新生成

3. **后端行为**:
   - 生成任务有 5 分钟超时窗口
   - 超时后自动标记为失败，释放资源

#### 5.2.4 场景3: 章节编辑

##### 正常流程

**步骤1: 选择章节**
1. 用户点击左侧章节列表中的某一章节
2. 该章节高亮显示
3. 中部编辑区加载章节内容
4. 右侧显示章节元数据

**步骤2: 编辑内容**
1. 用户直接在中部文本区编辑
2. 实时显示字数: `当前 87 字 (建议 50-100 字)`
3. 字数超出时文字变橙色警告
4. 编辑后章节标题出现「已编辑」标记
5. 自动保存: 停止输入 2 秒后自动保存

**步骤3: 编辑元数据**
1. 用户可在右侧修改:
   - 场景名称
   - 添加/删除/编辑对白
   - 为对白标注情绪
2. 修改后自动保存

**步骤4: 章节操作**

#### 5.2.5 场景4: 剧本确认与工位流转

##### 前置条件
- 剧本工位状态为 `completed`
- 至少有 1 个章节

##### 正常流程

**步骤1: 审核剧本**
1. 用户浏览并编辑完所有章节
2. 检查左侧章节列表，确认无遗漏

**步骤2: 确认剧本**
1. 用户点击顶部「确认剧本」按钮（或直接点击「进入主体工位」）
2. 系统执行校验:
   - ✅ 至少 1 个章节
   - ✅ 所有章节有内容
   - ⚠️ 警告但不阻止: 字数不在建议范围
3. 校验通过后，剧本工位锁定（状态 → `locked`）
4. 自动跳转到主体工位

**步骤3: 解锁编辑（如需返回修改）**
1. 用户可在任意工位点击面包屑导航回到剧本工位
2. 显示提示: `剧本已锁定，编辑将影响后续工位数据。是否解锁？`
3. 用户确认后:

   - 分镜工位数据标记为「待刷新」

```
                                              ↓ 生成成功

                                              ↑                    │
                                              └────解锁编辑────────┘
```

### 5.3 阶段2: 主体提取与资产生成 (AssetAgent)

#### 5.3.1 工位概述

主体工位负责从剧本中提取角色、场景、道具等视觉资产，并为每个资产生成图片素材。

**工位布局**:
- **左侧**: 资产分类面板（角色/场景/道具三个 Tab）
- **中部**: 资产卡片网格（每行 3-4 个卡片）
- **右侧**: 资产详情与生成面板（选中资产后显示）
- **顶部**: 操作栏（提取主体、批量生成、筛选等按钮）

---

#### 5.3.2 场景1: 主体提取

##### 前置条件
- 已进入主体工位
- 本集剧本状态为 `completed` 或 `locked`
- 主体工位状态为 `idle`

##### 正常流程

**步骤1: 触发提取**
1. 用户点击顶部「提取本集主体」按钮
2. 按钮变为禁用状态，显示 Loading
3. 顶部出现进度提示: `正在分析剧本，提取主体资产...`
4. 主体工位状态 → `extracting`

**步骤2: Agent 执行提取**
AssetAgent 依次执行:
1. **实体识别**: 扫描所有章节，识别命名实体
2. **去重合并**: 同一实体的不同称呼合并（如"晚卿"与"苏晚卿"）
3. **分类标注**: 判断类型（角色/场景/道具）
4. **描述生成**: 为每个资产生成 30-80 字中文视觉描述
5. **Prompt 生成**: 将描述翻译为英文图片生成 Prompt
6. **章节关联**: 记录资产出现的所有章节 ID

**步骤3: 提取完成**
1. 进度提示消失
2. 显示 Toast: `✓ 提取完成，发现 8 个角色、5 个场景、3 个道具`
3. 左侧分类面板显示数量徽标
4. 中部卡片网格加载所有资产，默认显示「角色」Tab
5. 每个资产卡片显示:
   - 占位图（灰色/渐变背景）
   - 资产名称
   - 资产类型图标
   - 状态徽标: `待生成`
6. 主体工位状态 → `ready`

##### Agent 提取规则详解

| 类型 | 识别条件 | 合并规则 | Prompt 要素 |
|-----|---------|---------|------------|
| **角色 (role)** | 有明确人名、出现 ≥2 次、有动作或对白 | 同名/昵称/别名合并 | 性别、年龄感、外貌（发型/肤色/身材）、服装、气质、标志性特征 |
| **场景 (scene)** | 有明确地名/建筑名、出现 ≥2 次 | 同地点不同时间合并 | 空间类型、规模、光照条件、主要元素、氛围、色调 |
| **道具 (tool)** | 有明确物品名、剧情关键、需视觉呈现 | 同物品合并 | 形状、尺寸、材质、颜色、特殊效果 |

##### 输出数据结构

```typescript
interface Asset {
  id: string;                    // UUID
  episodeId: string;             // 所属集数
  seriesId: string;              // 所属系列
  
  // === 基础信息 ===
  name: string;                  // 资产名称（如"苏晚卿"、"御书房"）
  aliases: string[];             // 别名列表（如["晚卿", "苏姑娘"]）
  type: 'role' | 'scene' | 'tool';
  description: string;           // 中文视觉描述 (30-80字)
  prompt: string;                // 英文图片 Prompt
  chapterIds: string[];          // 出现的章节 ID
  appearanceCount: number;       // 出现次数
  
  // === 角色专属字段 ===
  gender?: 'male' | 'female' | 'unknown';
  age?: string;                  // 如 "少年"、"中年"、"老年"
  voice?: string;                // 绑定的配音 ID
  faceLocked?: boolean;          // 面部锁定开关
  faceReferenceImageId?: string; // 面部参考图 ID
  
  // === 变体支持 ===
  parentAssetId?: string;        // 父资产 ID（变体时填充）
  variantName?: string;          // 变体名称（如"战损状态"、"红衣版"）
  isVariant: boolean;            // 是否为变体资产
  
  // === 生成状态 ===
  state: 'pending' | 'generating' | 'completed' | 'failed';
  images: AssetImage[];          // 图片版本列表
  selectedImageId?: string;      // 主版本图片 ID
  
  // === 元数据 ===
  createdAt: number;
  updatedAt: number;
  isShared: boolean;             // 是否为系列共享资产
  sourceAssetId?: string;        // 如从系列共享复制而来
}

interface AssetImage {
  id: string;
  url: string;
  thumbnailUrl: string;
  model: string;                 // 生成使用的模型
  prompt: string;                // 生成时的 Prompt（可能与资产 Prompt 不同）
  referenceImageId?: string;     // 参考图 ID
  isSelected: boolean;           // 是否为主版本
  createdAt: number;
  metadata: {
    width: number;
    height: number;
    seed?: number;
    steps?: number;
  };
}
```

#### 5.3.7 场景6: 工位门禁与流转

##### 门禁规则

分镜工位解锁条件:
```typescript
function canProceedToStoryboard(episodeId: string): boolean {
  const assets = getEpisodeAssets(episodeId);
  
  // 必须有至少 1 个资产
  if (assets.length === 0) return false;
  
  // 所有资产必须完成（状态为 completed）
  const allCompleted = assets.every(a => a.state === 'completed');
  
  // 所有资产必须有选定的主版本图片
  const allHasSelected = assets.every(a => 
    a.images.length > 0 && a.images.some(img => img.isSelected)
  );
  
  return allCompleted && allHasSelected;
}
```

### 5.4 阶段3: 分镜生成与分镜图生成 (ProductionAgent)

#### 5.4.1 工位概述

分镜工位负责将剧本章节拆解为独立镜头，并为每个镜头生成关键帧图片。

**工位布局**:
- **左侧**: 章节导航列表（显示每章节包含的镜头数）

- **顶部**: 操作栏（生成分镜表、批量生成图片、导入/导出等）

#### 5.4.6 场景5: 连戏检测与修复

##### 概述
连戏（Continuity）是指镜头之间的视觉连续性，包括角色服装、场景元素、光影方向等的一致性。

##### 自动检测规则

| 检测项 | 规则 | 警告级别 |
|-------|------|---------|
| 同景别连续 | 相邻镜头景别相同 | 建议 |
| 180度轴线 | 相邻镜头人物朝向反转 | 警告 |
| 服装不一致 | 同角色同场景服装描述不同 | 错误 |
| 时间不一致 | 同场景光线描述矛盾 | 警告 |
| 道具缺失 | 上镜头有、下镜头无 | 警告 |

##### 警告显示
1. 镜头行出现警告图标（⚠️）
2. 悬停显示具体警告内容
3. 右侧面板「连戏警告」区域列出所有警告
4. 用户可点击「忽略此警告」或「修复」

##### 修复流程
1. 用户点击某警告的「修复」
2. 系统自动修改相关字段使其一致
3. 或打开对比视图，让用户手动选择正确值

### 5.5 阶段4: 故事板与视频生成

#### 5.5.1 工位概述

故事板工位负责将分镜图片转化为视频片段，是从静态图像到动态影像的关键工位。

**工位布局**:
- **左上**: 监视器（Monitor）- 超大画幅预览区，支持播放视频、显示字幕
- **左下**: 镜头时间线（Timeline）- 横向滚动的镜头缩略图列表
- **右侧**: 生成控制台 - Prompt 编辑、模型选择、生成参数
- **顶部**: 操作栏 - 批量生成、导出预览视频等

---

#### 5.5.2 场景1: 进入故事板工位

##### 前置条件
- 分镜工位所有镜头已完成（都有选定的主版本图片）

##### 正常流程

**步骤1: 进入工位**
1. 用户点击顶部导航的「故事板」Tab
2. 系统加载所有分镜数据和图片
3. 左下时间线显示所有镜头缩略图

### 5.6 阶段5: 成片工位

#### 5.6.1 工位概述

成片工位提供多轨非线性编辑能力，用于装配视频、对白、音效、BGM 等元素。

**工位布局**:
- **上部**: 预览监视器（播放装配后的完整视频）
- **下部**: 多轨时间线
  - 视频轨: 所有镜头视频
  - 对白轨: 配音音频
  - 音效轨: 音效
  - BGM 轨: 背景音乐
- **右侧**: 素材库（可拖拽添加音效、BGM）
- **顶部**: 操作栏（导出、分辨率设置等）

### 5.7 全局异常处理规格

本节定义跨工位的通用异常处理策略。

#### 5.7.1 网络异常处理

| 异常类型 | 检测方式 | 系统行为 | 恢复策略 |
|---------|---------|---------|---------|
| 网络断开 | navigator.onLine | 顶部红色横幅: `网络已断开` | 自动重试队列 |
| 请求超时 | 超时阈值 | Toast: `请求超时` | 指数退避重试 |
| 服务器错误 (5xx) | HTTP 状态码 | Toast: `服务暂时不可用` | 30秒后重试 |
| 客户端错误 (4xx) | HTTP 状态码 | Toast: `请求错误: [原因]` | 不自动重试 |

##### 网络恢复后行为
1. 检测到网络恢复
2. 横幅变绿: `网络已恢复`
3. 自动执行待重试的请求
4. 3秒后横幅消失

#### 5.7.2 生成任务失败处理

| 任务类型 | 失败后状态 | UI 表现 | 恢复方式 |
|---------|-----------|---------|---------|
| 剧本生成 | `draft` | 按钮恢复可点击 | 手动重试 |
| 主体提取 | `idle` | 按钮恢复可点击 | 手动重试 |
| 资产图片生成 | `failed` | 卡片红色标记 | 单独重试或批量重试 |
| 分镜表生成 | `idle` | 按钮恢复可点击 | 手动重试 |
| 分镜图生成 | `failed` | 行红色标记 | 单独重试或批量重试 |
| 视频生成 | `failed` | 镜头红色标记 | 单独重试或批量重试 |
| 导出 | - | 显示失败原因 | 手动重试 |

##### 失败详情记录
每个失败任务记录:
- 失败时间
- 错误代码
- 错误信息
- 请求参数快照

可在「任务中心」查看历史失败任务并重试。

#### 5.7.3 配额与限流处理

| 配额类型 | 限制 | 提示时机 | 用户操作 |
|---------|-----|---------|---------|
| AI 调用次数 | 1000次/月 | 剩余 10% 时提醒 | 升级套餐 |
| 图片生成次数 | 500张/天 | 用尽时阻止 | 等待次日 |
| 视频生成次数 | 100条/天 | 用尽时阻止 | 等待次日 |
| 导出次数 | 20次/天 | 用尽时阻止 | 等待次日 |
| 存储空间 | 10GB/账户 | 80% 时提醒 | 清理或升级 |

##### 限流响应
当触发限流（HTTP 429）:
1. 显示 Toast: `请求过于频繁，请稍后重试`
2. 从响应头读取 `Retry-After`
3. 等待指定时间后自动重试
4. 批量任务暂停，显示「暂停中，等待限流解除」

#### 5.7.4 数据一致性与冲突处理

##### 乐观锁机制
- 每个实体有 `version` 字段
- 更新时携带当前版本号
- 版本不匹配时拒绝更新

##### 冲突处理流程
1. 用户 A 和用户 B 同时编辑同一资产
2. 用户 A 先保存成功
3. 用户 B 保存时版本冲突
4. 弹出冲突对话框:
   - 显示当前服务器版本（用户 A 的修改）
   - 显示本地版本（用户 B 的修改）
   - 选项: 「覆盖」/「放弃我的修改」/「合并」

##### 自动保存与离线编辑
1. 编辑内容每 5 秒自动保存到本地
2. 网络断开时，继续保存到本地
3. 网络恢复后，尝试同步到服务器
4. 如有冲突，触发冲突处理流程

#### 5.7.5 断点续传与恢复机制

##### 长任务断点续传
对于批量生成等长任务:
1. 任务分解为多个子任务
2. 每个子任务完成后标记 `done`
3. 中断后重新开始，跳过已完成的子任务

##### 页面刷新/关闭恢复
1. 页面关闭前保存状态到 localStorage:
   - 当前工位
   - 编辑中的内容
   - 进行中的任务 ID
2. 页面重新打开时:
   - 检测未保存的编辑 → 提示恢复
   - 检测进行中的任务 → 轮询状态

##### 会话恢复提示
```
检测到您之前的编辑会话尚未保存:
- 资产"苏晚卿"的描述修改
- 镜头 #12 的 Prompt 修改

是否恢复？
[恢复] [放弃]
```

### 6.2 单集生产全流程

#### 阶段1: 剧本工位

**目标**: 将小说原文转化为结构化剧本

| 步骤 | 用户操作 | 系统响应 | 状态变化 |
|-----|---------|---------|---------|
| 1 | 进入剧本工位 | 显示工位界面 | - |
| 2 | 粘贴/导入原文 | 校验原文长度 | `idle` → `draft` |
| 3 | 点击「生成剧本」 | ScriptAgent 执行 | `draft` → `generating` |
| 4 | 等待生成完成 | 章节逐个出现 | `generating` → `completed` |
| 5 | 审核/编辑章节 | 实时保存 | 保持 `completed` |
| 6 | 点击「确认剧本」 | 锁定剧本 | `completed` → `locked` |

**出口条件**: 剧本状态为 `locked`，解锁主体工位

### 6.4 典型异常场景处理

#### 场景: 生成失败后的恢复

**情况**: 批量生成 50 个分镜图，第 23 个失败

**处理流程**:
1. 系统标记失败的镜头为红色
2. 其余镜头继续生成
3. 全部完成后显示: `批量生成完成，成功 49 个，失败 1 个`
4. 用户点击失败镜头 → 查看失败原因
5. 修改 Prompt 或参数 → 重新生成

#### 场景: 跨工位返工

**情况**: 在分镜工位发现剧本需要修改

**处理流程**:
1. 用户可自由切换到剧本工位
2. 修改剧本内容
3. 系统提示: `剧本已修改，以下内容可能需要更新:`
   - 受影响的资产列表
   - 受影响的镜头列表
4. 用户选择:
   - 「重新提取主体」- 全部重来
   - 「增量更新」- 仅更新受影响部分
   - 「忽略」- 保持现有内容

#### 场景: 中途关闭浏览器

**处理流程**:
1. 系统定期保存工作状态 (5秒一次)
2. 用户关闭浏览器
3. 重新打开项目:
   - 进行中的生成任务: 恢复轮询状态
   - 未保存的编辑: 弹出恢复提示
   - 批量任务: 从断点继续

## 10. 逻辑一致性保障

本章节定义系统中容易产生冲突或不一致的关键场景，以及相应的处理机制。

### 10.1 门禁与编辑冲突处理

**问题**: 下游工位锁定后，上游资源需要修改怎么办？

#### 版本快照机制

```typescript
interface ScriptVersion {
  version: number;
  content: Chapter[];
  timestamp: number;
  isActiveVersion: boolean;   // 下游引用的版本
}

interface AssetVersion {
  version: number;
  images: ImageCandidate[];
  selectedImageId: string;
  timestamp: number;
  isActiveVersion: boolean;
}
```

#### 状态转换规则

| 当前状态 | 操作 | 结果 |
|---------|------|------|
| 剧本 `completed` + 分镜已生成 | 编辑剧本 | 创建新版本 `draft`，保留旧版本供下游继续使用 |
| 资产 `locked` | 编辑资产描述 | 允许编辑，提示 "资产已被 N 个分镜引用，修改后需重新生成相关分镜" |
| 分镜 `locked` | 编辑分镜 | 状态变为 `draft`，关联视频自动解除 selected 状态 |

#### 级联影响警告

```typescript
interface CascadeImpactWarning {
  sourceType: 'script' | 'asset' | 'shot';
  sourceId: string;
  impacts: {
    targetType: string;
    targetIds: string[];
    severity: 'info' | 'warning' | 'critical';
    message: string;
  }[];
}

// 示例: 编辑资产时的级联影响
{
  sourceType: 'asset',
  sourceId: 'asset_001',
  impacts: [
    {
      targetType: 'shot',
      targetIds: ['shot_003', 'shot_007', 'shot_012'],
      severity: 'warning',
      message: '3 个分镜引用了此资产，修改后建议重新生成分镜图'
    },
    {
      targetType: 'video',
      targetIds: ['video_003'],
      severity: 'critical',
      message: '1 个视频已使用此资产的旧版本生成，可能造成一致性问题'
    }
  ]
}
```

### 10.2 资产引用一致性

**问题**: 资产被删除或修改后，引用它的分镜如何处理？

#### 引用状态追踪

```typescript
interface Shot {
  // ... 其他字段
  associatedAssetIds: string[];
  
  // 引用状态 (系统自动计算)
  assetRefStatus: 'valid' | 'broken' | 'stale';
  brokenAssetRefs?: {
    assetId: string;
    reason: 'deleted' | 'no_image' | 'version_mismatch';
  }[];
}
```

#### 删除资产时的处理

```
用户点击删除资产
    ↓
系统检查引用情况
    ↓
  ┌─ 无引用 → 直接删除
  └─ 有引用 → 弹出警告对话框
         ├─ 显示引用该资产的镜头列表
         ├─ 选项1: 取消删除
         ├─ 选项2: 强制删除 (相关镜头标记为 broken)
         └─ 选项3: 替换为其他资产后删除
```

#### UI 状态显示

| 引用状态 | 视觉标记 | 可执行操作 |
|---------|---------|----------|
| `valid` | 绿色图标 | 正常生成 |
| `stale` | 黄色图标 + ⚠️ | 提示建议重新生成 |
| `broken` | 红色图标 + ❌ | 阻止生成，强制修复 |

### 10.3 并发写入保护

**问题**: 批量任务执行中，用户手动修改同一实体怎么办？

#### 乐观锁增强

```typescript
interface OptimisticLock {
  entityId: string;
  entityType: 'script' | 'asset' | 'shot' | 'video';
  version: number;                    // 数据版本号
  lastModifiedAt: number;
  lastModifiedBy?: string;
}
```

#### 批量任务参数快照

```typescript
interface BatchTask {
  id: string;
  type: 'generate_assets' | 'generate_shots' | 'generate_videos';
  items: BatchTaskItem[];
  createdAt: number;
  status: 'pending' | 'running' | 'completed' | 'partial_failed';
}

interface BatchTaskItem {
  targetId: string;                   // 目标实体 ID
  snapshotParams: Record<string, any>; // 创建任务时的参数快照
  paramVersion: number;               // 快照时的版本号
  status: 'pending' | 'running' | 'completed' | 'failed' | 'skipped';
  skipReason?: string;
}
```

#### 冲突检测流程

```
批量任务项准备执行
    ↓
比较 snapshotParams.version vs 当前 entity.version
    ↓
  ┌─ 版本一致 → 正常执行
  └─ 版本不一致 → 冲突处理
         ├─ 策略1 (默认): 跳过该项，记录 skipReason
         ├─ 策略2: 使用最新参数重新执行
         └─ 策略3: 暂停批量任务，等待用户决策
```

### 13.2 生成时连戏约束

**约束传递机制**:

```typescript
interface ContinuityContext {
  prevShot?: {
    shotId: string;
    imageUrl: string;
    dominantColors: string[];
    lightDirection?: 'left' | 'right' | 'front' | 'back';
  };
  nextShot?: {
    shotId: string;
    imageUrl?: string;
  };
  sceneContext: {
    sceneId: string;
    referenceImages: string[];    // 场景参考图
    lightingPreset?: string;
  };
  characterContext: {
    characterId: string;
    referenceImages: string[];    // 角色参考图
    faceLocked: boolean;
  }[];
}
```

**应用方式**:
1. 生成分镜图时，自动携带 `prevShot` 和相关角色参考图

### 13.3 连戏问题修复

## 14. 多模态输入能力

### 14.1 音频前置模式 (Audio-First)

**概念**: 先录制/生成对白音频，再基于音频时长和内容生成视频。

**工作流**:

```
步骤1: 输入对白文本
    ↓
步骤2: 选择角色语音 (TTS 或上传录音)
    ↓
步骤3: 生成/上传对白音频
    ↓
步骤4: 系统自动计算镜头时长 = 音频时长 + 缓冲
    ↓
步骤5: 生成与口型同步的视频 (Lip Sync)
```

**数据结构**:

```typescript
interface DialogueTrack {
  shotId: string;
  audioFile?: string;
  duration: number;             // 音频时长 (ms)
  transcript: string;           // 对白文本
  characterId?: string;         // 说话角色
  voiceId?: string;             // TTS 语音 ID
  lipSyncEnabled: boolean;      // 是否启用口型同步
}
```

### 14.2 图像输入增强

| 输入类型 | 用途 | 处理方式 |
|---------|------|---------|
| 手绘草图 | 分镜构图参考 | ControlNet 线稿模式 |
| 真人照片 | 角色参考 | 提取面部特征，锁定一致性 |
| 场景照片 | 场景参考 | 提取风格/色调/构图 |

## 16. 关键业务规则

1. 系列级规则默认继承到单集。
2. 单集可临时覆盖，但不能静默污染系列级规则。
3. 主体资产分为"本集主体"、"系列共享主体"和"全局资产"。
4. 系列共享主体必须具备主版本。
5. 已锁定主体、分镜、视频应有明确视觉标记。
6. 故事板与成片只能使用已选版本 (Selected Take) 进入下游。
7. 审校未通过不得进入导出。
8. 开启"面部锁定 (Face Locked)"的角色资产，在后续分镜生成中必须强制应用一致性约束（如垫图、LoRA 或 Reference ID）。
9. 音频前置模式下，镜头时长由绑定的对白/音效长度自动锁定，手动修改需解除绑定。
10. 数据飞轮与私有化模型：当系列累积足够多的 Selected 优质镜头后，系统可提示用户训练该系列专属的风格微调模型 (Custom LoRA)。
11. 资产删除前必须检查引用关系，强制删除将导致相关分镜标记为 `broken`。
12. 批量任务执行前快照参数，执行时检测版本冲突，冲突项默认跳过。
13. 上游数据修改时，系统自动计算级联影响并提示用户。
14. 全局资产导入到系列后，可选择保持同步或独立副本。
15. 生成任务预估成本超过阈值时，必须用户确认后才能执行。

## 19. 无限画布模块设计

### 19.1 概述

无限画布（Infinite Canvas）是单集工作台的全局可视化视图，以空间化的方式展示章节、主体、分镜三类节点及它们之间的关系，并集成 AI 助手以支持直接在画布上进行内容修改。

入口：单集工作台顶部 Tab 栏中的「画布」按钮，点击后以全屏覆盖层方式打开，左侧侧边导航栏（240px）保持可见。

### 19.2 布局结构

```
┌──────────────────────────────────────────────────────────────┐
│  [顶部控制栏]  标题 · 无限画布  [过滤器] [缩放] [AI助手] [×]  │
├─────────────────────────────────────────┬────────────────────┤
│                                         │                    │
│  [节点详情面板]   [无限画布视口]         │  [AI 助手聊天面板] │
│   280px 可收起        ∞                 │     340px 固定     │
│                                         │                    │
└─────────────────────────────────────────┴────────────────────┘
```

- **整体定位**: `position: fixed; left: 240px; inset: 0;` 覆盖主内容区，Z 层级 50
- **节点详情面板**: 点击节点后从左侧滑入（280px），带 CSS `transition-all duration-200`
- **画布视口**: 弹性占满剩余宽度，支持鼠标拖拽平移、滚轮缩放
- **AI 聊天面板**: 右侧固定宽度 340px，可通过顶栏按钮收起/展开

### 19.3 节点类型

| 节点类型 | 对应数据 | 主色 | 最小显示字段 |
|---------|---------|------|------------|
| **ChapterNode** | Chapter | 紫色 (violet) | 序号、标题、场景、内容摘要 |
| **AssetNode** | Asset | 天蓝 (sky) | 名称、类型、状态、缩略图 |
| **ShotNode** | Shot | 琥珀 (amber) | 序号、标题、景别、时长、缩略图 |
| **DiffNode** | AI Proposal | 绿色 (emerald) | 变更摘要、字段 diff、Keep/Discard |

所有节点：
- 卡片宽度约 220px，高度根据内容自适应
- Header 带 grip 图标，按住可拖动节点
- 点击节点 body 打开左侧详情面板
- 选中状态：边框亮色高亮

### 19.4 画布交互

#### 19.4.1 视口变换
- **平移（Pan）**: 在空白区域按住鼠标左键拖拽，`transform: translate(tx, ty)`
- **缩放（Zoom）**: 鼠标滚轮，范围 20%–200%，步长 15%
- **适配视图**: 顶栏「适配视图」按钮，自动计算 scale/translate 使所有节点完整可见
- 缩放/平移状态存储在 `transform: { x, y, scale }` state

#### 19.4.2 节点拖拽
- 拖拽触发：鼠标按下节点 header 的 grip 区域（`onMouseDown` → `draggingRef`）
- 坐标换算：`nodeDelta = mouseDelta / transform.scale`（保证缩放下拖拽精确）
- 拖拽位置存储在独立的 `nodePositions` map，覆盖自动布局坐标
- 释放鼠标（`onMouseUp` / `onMouseLeave`）结束拖拽

#### 19.4.3 自动布局引擎
初始布局由 `buildCanvasLayout()` 计算，将三类节点分列排布：

```
Col 0: ChapterNodes (y 步长 300px)
Col 1: AssetNodes   (y 步长 280px)
Col 2: ShotNodes    (y 步长 260px，每列满 N 个后新起一列)
```

坐标为逻辑坐标，通过 CSS transform 映射到屏幕空间。

#### 19.4.4 节点过滤
顶栏三个 Toggle 按钮分别控制章节、主体、分镜节点的可见性，至少保留一类可见。

### 19.5 AI 助手聊天面板

#### 19.5.1 请求/响应模式
- **非流式 JSON**：`POST /api/canvas-chat` 返回 `{ reply: string; proposals: Proposal[] }`
- AI 分析上下文（当前集所有章节/主体/分镜）后：
  - 若为问答：直接在 reply 中回答
  - 若需修改内容：通过工具调用生成 Proposal，**不立即 dispatch**，等用户确认

#### 19.5.2 工具（Tools）

| 工具 | 作用 | 入参 |
|-----|------|------|
| `updateChapter` | 修改章节内容/场景 | chapterId, 变更字段 |
| `updateAsset` | 修改主体描述/Prompt | assetId, 变更字段 |
| `updateShot` | 修改分镜字段 | shotId, 变更字段 |
| `generateShotImages` | 触发分镜图片重新生成 | shotIds |

工具返回 Proposal 对象，不直接写库。

#### 19.5.3 Proposal 数据结构

```typescript
interface Proposal {
  proposalId: string;         // 唯一 ID
  kind: string;               // 'updateChapter' | 'updateAsset' | 'updateShot' | ...
  anchorNodeId: string;       // 关联节点 ID（用于定位 DiffNode 位置）
  args: Record<string, unknown>; // 工具调用参数（Keep 时直接 dispatch）
  diff: Array<{
    field: string;
    before: string;
    after: string;
  }>;
  summary: string;            // 一句话变更摘要
  position?: { x: number; y: number }; // 画布上的显示位置（运行时填充）
}
```

### 19.6 DiffNode（AI 建议节点）

参考 Cursor 的代码 diff 交互设计，当 AI 提出内容修改建议时：

1. 在锚点节点右侧 320px 处生成 **DiffNode**
2. DiffNode 显示：
   - 标题：变更摘要（`summary`）
   - 变更类型 badge
   - 字段 diff 列表（`before` → `after`，删除线 / 绿色高亮）
   - **Keep（保留）** 按钮：`POST /api/studio` dispatch 实际命令，移除 DiffNode
   - **Discard（撤销）** 按钮：直接移除 DiffNode，不做任何修改
3. DiffNode 同样可拖拽（`proposal:id` 前缀区分拖拽目标）
4. 顶栏显示待确认数量 badge：`N 条 AI 建议待确认`

### 19.7 节点详情面板

点击任意节点（ChapterNode / AssetNode / ShotNode）后，从画布左侧滑入 280px 详情面板，展示完整字段：

| 节点类型 | 展示内容 |
|---------|---------|
| **ChapterNode** | 标题、场景、正文全文、对白列表（speaker + content） |
| **AssetNode** | 封面图及所有版本图、名称/类型/状态、描述、prompt（mono）、配音 |
| **ShotNode** | 封面图及所有 Take 缩略图、景别/运镜/时长、场景描述、对白（蓝）、音效（紫）、prompt（mono）、关联主体 chips |

点击面板右上角 X 或再次点击同节点关闭面板。
