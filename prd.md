# Mercruiser 重构版业务 PRD

## 文档信息
- 文档名：Mercruiser 重构版业务 PRD
- 版本：v2.1
- 状态：Draft
- 目标版本：Web Studio 重构版
- 适用范围：系列化 AI 短剧 / 漫剧生产
- 更新说明：v2.1 新增完整的 Agent 驱动生产工作流章节 (第5章)，详细定义了从原文导入到成片导出的 Agent 职责、数据结构和门禁规则

## 1. 文档目的

本 PRD 用于定义 Mercruiser 下一阶段的核心业务模型、生产工作流和产品范围。

本次重构的核心目标不是简单优化页面，而是将产品从“阶段式工作台”升级为“系列治理 + 单集工业化生产线”的双层体系。

## 2. 产品定义

Mercruiser 是一个面向 AI 短剧 / 漫剧生产的 Studio。

其本质不是单次生成工具，而是一个持续生产系统，包含：

- 系列治理层
- 单集生产线层
- Agent 编排层
- 任务与系统治理层

## 3. 产品定位

### 3.1 系列原生
产品以“系列”为顶层对象，以“集数”为交付单元，以“章节 / 镜头”为生产颗粒度。

### 3.2 工业化生产线
单集内部不是一个大而全工作台，而是一条明确生产线：

1. 概览
2. 剧本
3. 主体
4. 分镜
5. 故事板
6. 成片

### 3.3 Agent 编排驱动
Agent 不只是聊天工具，而是生产协调器，负责：

- 当前状态识别
- 缺口诊断
- 下一步建议
- 任务编排
- 失败恢复建议
- 质量提示

### 3.4 资产沉淀与复用
每一集生产不是孤立行为，高价值资产、风格规则、提示词经验必须沉淀回系列层。

## 4. 核心业务结构

### 4.1 系列层
负责：

- 系列基础信息
- 世界观 / 风格 / 导演规则
- 系列共享主体资产
- 系列级模型 / Prompt / Agent 策略
- 集数列表与总进度
- 系列级风险管理

### 4.2 集数层
负责：

- 单集源内容
- 单集章节结构
- 单集镜头结构
- 单集故事板
- 单集成片
- 单集导出结果

### 4.3 章节层
章节是剧本工位的工作单位，负责：

- 承载文本内容
- 控制字数与节奏
- 作为后续主体提取、分镜拆解的输入源

### 4.4 镜头层
镜头是分镜工位的工作单位，负责：

- 场景
- Prompt (提示词)
- 景别与构图
- 运镜 (Camera Motion)
- 光影
- 对白
- 音效
- 时长 (支持音频前置驱动的自动计算)
- 分镜描述
- Take (候选条) 管理
- 连戏状态 (Continuity Status)

### 4.5 故事板层
故事板是镜头的多模态装配层，负责：

- 镜头图
- 参考主体
- 分镜素材
- 镜头视频候选
- 选定版本

### 4.6 Agent 编排层
横跨全局，负责：

- 阶段推进
- 前置条件校验
- 推荐动作
- 批量动作触发
- 失败恢复建议

## 5. Agent 驱动的生产工作流 (核心)

本章节详细定义 Agent 在各工位的具体职责、触发条件、输入输出和前置依赖。

### 5.1 工作流总览

```
小说原文导入
    ↓
[阶段1: 剧本生成] → ScriptAgent (手动触发)
    ├─ 输入: 小说原文 + 系列设定
    ├─ 输出: 按集数拆分的剧本章节
    └─ 产物: episodes[].chapters[]
    ↓
[阶段2: 主体提取] → AssetAgent (手动触发)
    ├─ 输入: 本集剧本章节
    ├─ 输出: 角色/场景/道具资产列表
    ├─ 产物: assets[] (含 name, description, prompt, type)
    └─ 门禁: 剧本完成后才可触发
    ↓
[阶段3: 资产生成] → 用户触发 (单个/批量)
    ├─ 输入: asset.prompt + 模型配置
    ├─ 输出: 资产图片 (多版本)
    ├─ 产物: asset.images[] + selectedVersion
    └─ 门禁: 提取完成后才可生成
    ↓
[阶段4: 分镜生成] → ProductionAgent (手动触发)
    ├─ 输入: 本集剧本 + 已生成资产列表
    ├─ 输出: 分镜表 (Shot List)
    ├─ 产物: shots[] (完整12字段结构)
    └─ 门禁: 本集所有资产必须已生成图片
    ↓
[阶段5: 分镜图生成] → 用户触发 (单个/批量)
    ├─ 输入: shot.prompt + 关联资产参考图
    ├─ 输出: 分镜图片 (多版本)
    ├─ 产物: shot.images[] + selectedVersion
    └─ 门禁: 分镜表生成完成
    ↓
[后续阶段: 故事板 → 成片 → 导出]
```

### 5.2 阶段1: 剧本生成 (ScriptAgent)

#### 触发方式
- **手动触发**: 用户在剧本工位点击"从原文生成剧本"按钮
- **输入方式**: 粘贴文本 / 导入文件 (.txt, .docx)

#### Agent 职责
1. **内容分析**: 解析小说原文结构、人物关系、场景描写
2. **集数拆分**: 按叙事节奏将原文拆分为多集
3. **章节生成**: 每集生成多个章节，保持 50-100 字/章节的颗粒度
4. **对白提取**: 识别并标记对白内容
5. **场景标注**: 为每个章节标注主要场景

#### 输出结构
```typescript
interface GeneratedScript {
  episodeId: string;
  chapters: {
    id: string;
    index: number;
    content: string;         // 章节文本 (50-100字)
    scene: string;           // 场景名称
    dialogues: {             // 对白列表
      speaker: string;       // 说话人
      content: string;       // 对白内容
    }[];
  }[];
}
```

#### 前置条件
- 小说原文已导入
- 系列设定 (世界观、风格) 已配置

#### 后置状态
- 剧本工位状态 → `completed`
- 可触发: 主体提取

---

### 5.3 阶段2: 主体提取 (AssetAgent)

#### 触发方式
- **手动触发**: 用户在主体工位点击"提取本集主体"按钮

#### Agent 职责
1. **实体识别**: 从剧本中识别所有角色、场景、道具
2. **去重合并**: 合并重复出现的实体，建立统一名称
3. **描述生成**: 为每个资产生成 30-80 字的视觉描述
4. **Prompt 生成**: 为每个资产生成英文图片生成提示词
5. **分类标注**: 标注资产类型 (role / scene / tool)
6. **关联章节**: 记录资产出现的章节ID列表

#### 资产数据结构
```typescript
interface Asset {
  id: string;
  name: string;              // 资产名称 (如 "苏晚卿", "大殿")
  type: 'role' | 'scene' | 'tool';  // 资产类型
  description: string;       // 视觉描述 (30-80字)
  prompt: string;            // 英文图片生成提示词
  chapterIds: string[];      // 出现的章节ID列表
  
  // 生成后填充
  images: {                  // 图片版本列表
    id: string;
    url: string;
    isSelected: boolean;     // 是否为主版本
    createdAt: number;
  }[];
  state: 'pending' | 'generating' | 'completed' | 'failed';
  
  // 角色专属
  voice?: string;            // 绑定的配音ID
  faceLocked?: boolean;      // 是否开启面部锁定
  
  // 派生资产支持 (如不同服装状态)
  parentAssetId?: string;    // 父资产ID
  variantName?: string;      // 变体名称 (如 "战损状态")
}
```

#### 资产类型详细定义

| 类型 | 描述 | 提取规则 | Prompt 要求 |
|-----|------|---------|------------|
| **role** (角色) | 有名字的人物 | 有明确名字、反复出现、参与对话或动作 | 外貌特征、服装、气质、姿态 |
| **scene** (场景) | 地点/环境 | 有明确名称、多次出现 | 空间结构、光照、关键元素、氛围 |
| **tool** (道具) | 重要物品 | 剧情关键道具、需特写镜头 | 形状、颜色、材质、特效 |

#### 前置条件
- 本集剧本已生成完成
- 剧本状态 = `completed`

#### 后置状态
- 主体工位状态 → `ready` (待生成图片)
- 记录: 本集依赖的资产ID列表

---

### 5.4 阶段3: 资产图片生成 (用户触发)

#### 触发方式
- **单个生成**: 点击资产卡片的"生成"按钮
- **批量生成**: 勾选多个资产 → 点击"批量生成"

#### 生成流程
1. **Prompt 确认/编辑**: 用户可修改 Agent 生成的 Prompt
2. **模型选择**: 选择图片生成模型
3. **参考图上传** (可选): 上传角色参考图
4. **触发生成**: 调用图片生成 API
5. **版本管理**: 支持多次生成，保留所有版本
6. **选定主版本**: 用户选择一张作为主版本

#### 门禁检查
分镜生成前，系统自动检查:
```typescript
function canProceedToStoryboard(episodeId: string): boolean {
  const assets = getEpisodeAssets(episodeId);
  return assets.every(asset => 
    asset.state === 'completed' && 
    asset.images.some(img => img.isSelected)
  );
}
```

- ✅ 所有资产状态 = `completed`
- ✅ 所有资产有选定的主版本图片
- ❌ 否则: 分镜按钮置灰，提示"请先完成所有资产生成"

### 5.5 阶段4: 分镜生成 (ProductionAgent)

#### 触发方式
- **手动触发**: 用户在分镜工位点击"生成分镜表"按钮
- **门禁**: 本集所有资产必须已生成且选定主版本

#### Agent 职责

##### 第一步: 导演规划 (Director Planning)
基于系列设定生成6维导演规划:
1. **主题立意与叙事核心**: 核心主题、情感主线、表达策略
2. **视觉风格与画面基调**: 色调、质感、构图风格、光影体系
3. **叙事结构与节奏规划**: 段落划分、情绪曲线、转折点
4. **分场景情绪与画面意图**: 情绪目标、氛围方向、镜头意图
5. **声音与音乐方向**: 音乐风格、环境音、沉默运用
6. **转场与视觉连续性**: 转场策略、过渡手法、连续锚点

##### 第二步: 分镜表生成 (Storyboard Table)
将剧本拆分为独立镜头，每个镜头包含完整12字段:

#### 分镜数据结构
```typescript
interface Shot {
  id: string;
  episodeId: string;
  index: number;                    // 镜头序号
  
  // === 核心12字段 ===
  description: string;              // 画面描述 (15-50字，前中后景)
  scene: string;                    // 场景名称
  associatedAssetIds: string[];     // 关联资产ID列表
  associatedAssetNames: string[];   // 关联资产名称列表
  duration: number;                 // 时长 (秒，最大8秒)
  shotSize: ShotSizeEnum;           // 景别
  cameraMove: string;               // 运镜
  action: string;                   // 角色动作
  emotion: string;                  // 情绪 (2-10字)
  lighting: string;                 // 光影氛围 (5-40字)
  dialogue: string;                 // 台词
  sound: string;                    // 音效
  
  // === 生成字段 ===
  prompt: string;                   // 图片生成提示词 (含 @图N 资产引用)
  videoDesc: string;                // 视频描述 (结构化文本)
  
  // === 状态字段 ===
  state: 'draft' | 'ready' | 'generating' | 'completed' | 'failed';
  images: {
    id: string;
    url: string;
    isSelected: boolean;
  }[];
  
  // === 轨道分组 (用于多参视频) ===
  track: string;
  trackId: string;
}

// 景别枚举
type ShotSizeEnum = 
  | '大远景'    // Extreme Wide Shot
  | '远景'      // Wide Shot  
  | '全景'      // Full Shot
  | '中景'      // Medium Shot
  | '近景'      // Medium Close-up
  | '特写'      // Close-up
  | '大特写';   // Extreme Close-up
```

#### 分镜生成规则

##### 时长规则
| 镜头类型 | 建议时长 | 说明 |
|---------|---------|------|
| 特写对白 | 3-5秒 | 角色近景对白 |
| 全身镜头 | 3-5秒 | 角色全身展示 |
| 动作镜头 | 2-4秒 | 动作序列 |
| 远景/场景 | 3-5秒 | 场景建立 |
| 复杂场景 | 5-8秒 | 多人/多动作 |
| **单镜头上限** | **8秒** | 超过需拆分 |

##### 拆分规则
需要新镜头的情况:
- 场景/地点变换
- 时间跳跃
- 主体变更
- 景别大幅变化
- 关键动作节点

不需要新镜头的情况:
- 连续对话 (同景别)
- 微表情/微动作变化

##### 视觉连续性规则 (铁律)
1. **动作连续**: 上镜头动作结束点 = 下镜头开始点
2. **景别递进**: 避免同景别连续切换，应渐进推近/拉远
3. **视线一致**: 180度轴线规则
4. **空间逻辑**: 人物朝向、位置关系合理
5. **信息控制**: 每镜头传递特定信息
6. **节奏密度**: 动作/事件数量匹配时长
7. **安全区**: 镜头首尾0.5秒预留转场

#### Prompt 生成规则
从分镜字段 → 图片 Prompt 的映射:

| 分镜字段 | → Prompt 组成 | 说明 |
|---------|--------------|------|
| 画面描述 | 核心视觉语言 | 转为镜头语言 |
| 景别 | 镜头参数 | medium shot, close-up 等 |
| 场景 | 背景描述 | 场景纹理约束词 |
| 角色动作 | **视频首帧状态** | "手触及剑柄" 而非 "已拔剑" |
| 情绪 | 表情词 | 情绪→表情映射表 |
| 光影氛围 | 光照描述 | 方向+色温+亮度 |
| 运镜/台词/音效 | 不入Prompt | 仅作元数据 |

##### 资产引用绑定 (@图N 语法)
```
@图1为古装少女{role} @图2为宫殿{scene}，中景构图，@图1站在@图2前，手持花枝侧身而立，
神情温柔，眼神专注柔和。冷调侧光打亮面部轮廓，朱红宫墙营造典雅氛围。
```
- 所有资产在 Prompt 中用 `@图N` 引用
- 生成时用资产主版本图片作为参考图
- 保持角色/场景视觉一致性

#### 前置条件
- 本集剧本完成
- 本集所有资产已生成且选定主版本
- `canProceedToStoryboard(episodeId) === true`

#### 后置状态
- 分镜工位状态 → `ready`
- 可触发: 分镜图片生成

### 5.6 阶段5: 分镜图片生成 (用户触发)

#### 触发方式
- **单个生成**: 点击分镜卡片的"生成图片"按钮
- **批量生成**: 勾选多个分镜 → 点击"批量生成"

#### 生成流程
1. **Prompt 确认/编辑**: 用户可修改 Agent 生成的 Prompt
2. **资产关联确认**: 检查 @图N 引用的资产是否正确
3. **模型选择**: 选择图片生成模型
4. **触发生成**: 
   - 解析 @图N 引用
   - 加载对应资产主版本图片作为参考
   - 调用图片生成 API (ControlNet / Character Reference)
5. **版本管理**: 支持多次生成"抽卡"，保留所有版本
6. **选定主版本**: 用户选择最佳版本进入故事板

#### 生成模式
| 模式 | 说明 | 适用场景 |
|-----|------|---------|
| **图辅助模式** (默认) | 生成分镜关键帧图片 | 需要视觉预览 |
| **纯文本模式** | 仅生成 videoDesc，不生成图片 | 多参视频直出 |
| **首帧模式** | 每镜头独立生成，不分组 | 高质量单镜头 |

                                    ↓
故事板 → 成片 → 导出

## 6. 标准业务主线

### 6.1 系列建立
1. 创建系列
2. 配置系列设定 (世界观、风格、导演规则)
3. 配置系列策略 (默认模型、Prompt 模板)
4. 导入或创建集数

### 6.2 单集生产 (详细步骤)

#### Step 1: 导入原文 → 生成剧本
1. 进入单集 → 剧本工位
2. 粘贴/导入小说原文
3. 点击"从原文生成剧本" → ScriptAgent 执行
4. 等待生成完成 → 审核/编辑章节内容
5. 确认剧本 → 解锁主体工位

#### Step 2: 提取主体 → 生成资产图
1. 进入主体工位
2. 点击"提取本集主体" → AssetAgent 执行
3. 等待提取完成 → 审核/编辑资产列表
4. 对每个资产: 编辑 Prompt → 单个或批量生成图片
5. 为每个资产选定主版本图片
6. 所有资产完成 → 解锁分镜工位

#### Step 3: 生成分镜 → 生成分镜图
1. 进入分镜工位
2. 点击"生成分镜表" → ProductionAgent 执行
3. 等待生成完成 → 审核/编辑分镜表 (12字段)
4. 对每个分镜: 编辑 Prompt → 单个或批量生成图片
5. 为每个分镜选定主版本图片
6. 所有分镜完成 → 进入故事板

#### Step 4: 故事板 → 成片 → 导出
1. 进入故事板工位 → 生成镜头视频
2. 进入成片工位 → 剪辑与装配
3. 导出前确认 → 导出结果

### 6.3 反哺系列
1. 将高价值主体升级为系列共享资产
2. 将有效策略沉淀为系列 Prompt / Agent 策略
3. 将失败经验沉淀为恢复建议

## 7. 用户角色

### 9.1 系列制片 / 内容统筹
关注系列总进度、设定统一性、资源复用率和交付节奏。

### 9.2 集数导演 / 创作执行者
关注单集从文本到成片的推进效率。

### 9.3 Prompt / 流程调优操作者
关注模型、提示词、Agent 策略和稳定性。

### 9.4 系统配置管理员
关注供应商、模型、请求、数据与运行环境。

## 8. 功能范围

### 9.1 工作区首页

- 快速进入最近生产中的系列

### 9.2 系列详情

- 共享主体资产
- 系列策略配置

### 9.3 单集执行页
必须重构为以下固定页签：

- 概览
- 剧本
- 主体
- 分镜
- 故事板
- 成片

### 9.4 剧本工位
必须支持：

- 左侧章节列表
- 中部章节文本编辑
- 右侧全局设定侧边栏 (包含视频比例、创作模式、风格参考等)
- 音频前置生成 (Audio-First Workflow)：先生成全局旁白/对白音频
- 一键拆解分镜 (Breakdown to Shots) 与资产提取
- 新增章节
- 章节顺序管理

### 9.5 主体工位
必须支持：

- 角色 / 场景 / 道具分类
- 主体卡片列表 (支持绑定 Voice 配音选择)
- 主体描述编辑
- 资产状态管理 (Stateful Assets)：支持同一角色的多状态管理（如默认、战损、特定服装）
- 资产图片版本管理 (Asset Versioning)：支持多次生成资产图片，保留历史版本，并可随时切换主版本
- 右侧资产生成专属工作区 (支持提示词编辑、模型选择、参考图上传及生成素材池)
- 主版本选择
- 面部特征锁定 (Face Locking) 及一致性视觉标识
- 升级为系列共享资产

### 9.6 智能分镜表工位 (Shot List)
必须支持：

- 垂直列表布局，多列网格数据结构展示
- 镜头表结构化编辑 (包含场景、景别、光影、Prompt 提示词、对白、时长等)
- 运镜控制 (Camera Motion，如推拉摇移)
- 新增 / 删除 / 复制 / 重排镜头
- 分镜表导入导出

### 9.7 故事板工位 (Storyboard)
必须支持专业级视觉生成工作台布局：

- 左上监视器 (Monitor)：超大画幅预览，支持字幕编辑与音频试听
- 左下镜头时间线 (Timeline)：横向滚动缩略图列表，支持快速切换与状态查看，集成 AI 连戏检测警告 (Continuity Check)
- 右侧生成控制台：包含图/视频切换、当前镜头 Prompt 编辑、模型选择
- 参考主体 (Reference Subjects) 快速关联 (支持选择特定资产状态)
- Take (条) 与版本管理：支持图片与视频的多次生成抽卡，保留独立的图片候选池与视频候选池，并支持主版本选定 (Mark as Selected)
- 批量生成图片 / 视频

### 9.8 成片工位
必须支持：

- 多轨非线性编辑 (NLE) 时间线
- 视频轨、对白轨 (Dialogue)、音效/BGM轨对齐与装配
- 顺序调整与片段裁剪 (Scissors)
- 分辨率/帧率设置与全局预览
- 片段替换
- 导出前确认

### 9.9 任务中心

- 筛选任务
- 查看失败原因
- 重试
- 回溯到对应工位

### 9.10 设置中心

- 供应商

- Prompt
- Skill
- Memory
- Agent
- 请求
- 文件

## 10. 关键业务规则

1. 系列级规则默认继承到单集。
2. 单集可临时覆盖，但不能静默污染系列级规则。
3. 主体资产分为“本集主体”和“系列共享主体”。
4. 系列共享主体必须具备主版本。
5. 已锁定主体、分镜、视频应有明确视觉标记。
6. 故事板与成片只能使用已选版本 (Selected Take) 进入下游。
7. 审校未通过不得进入导出。
8. 开启“面部锁定 (Face Locked)”的角色资产，在后续分镜生成中必须强制应用一致性约束（如垫图、LoRA 或 Reference ID）。
9. 音频前置模式下，镜头时长由绑定的对白/音效长度自动锁定，手动修改需解除绑定。
10. 数据飞轮与私有化模型：当系列累积足够多的 Selected 优质镜头后，系统可提示用户训练该系列专属的风格微调模型 (Custom LoRA)。
